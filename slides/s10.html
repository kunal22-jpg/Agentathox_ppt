Engineering Challenges

We Broke Things. We Fixed Them. Here's What We Learned.

Status

5 Major Obstacles Solved

01

The Problem

Sub-800ms Voice Latency

Standard API chains (Whisper + GPT) took 2.3–3.1s round-trip. Felt robotic.

The Solution

Switched to full streaming architecture: Deepgram Nova-2 + Groq + Aura. Reduced to ~630ms total.

Streaming Websockets

Key Lesson

For conversational AI, streaming > request/response.

02

The Problem

LLM Rate Limits (429)

Groq's free tier RPM limits caused crashes during peak usage testing.

The Solution

Built backend auto-fallback. Retries via OpenAI GPT-3.5 in ~50ms if Groq fails. Invisible to user.

Multi-Provider Redundancy

Key Lesson

Never rely on a single LLM provider in production.

03

The Problem

Multimodal Storage

Diary video/audio files exceeded MongoDB Atlas free tier limits instantly.

The Solution

Decoupled storage. Media to Cloudinary (CDN), only URLs stored in MongoDB. Lean & fast.

Cloudinary API

Key Lesson

Databases are for data. CDNs are for media.

04

The Problem

WebGL on Low-End Android

3D workout gallery ran at 2-5fps on budget devices (target demographic).

The Solution

Implemented capability check. Auto-switches to CSS3 carousel if GPU memory is low.

Progressive Enhancement

Key Lesson

Your coolest feature shouldn't exclude your core users.

05

The Problem

Agentic State Management

Linear LangChain flows broke when complex diet/budget logic required looping.

The Solution

Rebuilt with LangGraph stateful nodes. Enables planning, self-correction, and loops.

LangGraph State Machine

Key Lesson

True agents need stateful graphs, not just chains.
